Senior-level HTML interview questions (5-6 years)

1) How do you design HTML structure for a large, content-driven site to support SEO, accessibility, and performance?
- Answer: Use semantic landmarks, unique titles/meta, server-side rendering/prerender for critical content, lazy-loading media, and consistent canonical URLs. Design for component reusability and progressive enhancement.

2) How do you handle metadata that varies per route in a Single Page App?
- Answer: Use server-side rendering or runtime APIs (e.g., Next.js Head) to set deterministic metadata on navigation; fallback to client-side updates with proper history management.

3) How to balance semantics vs ARIA when building complex widgets?
- Answer: Prefer native elements; use ARIA only to fill gaps. Follow WAI-ARIA Authoring Practices and ensure keyboard behavior mirrors native widgets.

4) How would you structure HTML to optimize for Core Web Vitals?
- Answer: Minimize render-blocking resources in head, inline critical CSS, preload critical assets, defer non-critical scripts, and optimize images.

5) How to design forms for resilience and accessibility at scale?
- Answer: Use proper labels/fieldsets, server-side validation, consistent error messaging linked with aria-describedby, and manageable state for progressive enhancement.

6) How to approach internationalization (lang, dir, metadata) in HTML?
- Answer: Set lang attributes per document/fragment, handle direction via dir, provide localized metadata and hreflang links for multi-region support.
